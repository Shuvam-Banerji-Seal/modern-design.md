How the pipeline works

Every entry goes through a two-phase pipeline. The lead agent dumps the live site (HTML, CSS, JS, fonts, images, SVGs, 3D models) into a gitignored tmp/<slug>/ folder. A sub-agent reads the dump and writes the design.md. A second sub-agent then reads the design.md alone and produces a working HTML/CSS/JS sample — proving the spec is complete enough to rebuild from.

Phase 1 — Dump

tools/scrape.py --use-playwright --depth 2 walks the live site, downloads every asset into tmp/<slug>/, and writes a manifest.json the sub-agent consumes.

Phase 2 — Specify

A sub-agent reads the dump and writes websites/<slug>/design.md following a strict template: Visual Language, Components, JS & Libraries, Animations Catalog, Assets, Motion, Accessibility.

Phase 3 — Reproduce

Another sub-agent reads only the design.md and produces showcase/html/<slug>/index.html + style.css + script.js — a working sample that embodies the spec.

Deploy

GitHub Actions builds the manifest from websites/, then publishes showcase/ to GitHub Pages on every push to main.
